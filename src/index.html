Continuous Integration!

I initially had this site synchronized with my S3 Bucket using the dandelion tool. However that meant that I had to execute a separate command to deploy the code. 

git add index.html
git commit -m "Added great content about the new work I am doing"
git push
dandelion deploy #Grrrrr
While one more command may not appear to be a lot. It is a 25% increase. That increase is also very risky as it is for a tool that is not part of my normal tool-chain. I know I will forget how to use it with any pause in use. Finally, automation is a first order feature of any product. I cannot automate the git commands, but I can automate deployments.

At first I search in vain for a tool that would work for me. My initial path was a complicated GitHub webhook to an AWS Lambda function. While that sounds cool and I would have learned a lot, my gut told me that was way too complicated. I also looked at using CodeDeploy and a lot of other dead ends. Eventually my GoogleFu was successful after a tweet begging for help.

I want to push files on commit to @github repo to @awscloud S3. I cannot find an existing solution & I must be using the wrong search term.

— Jacob Tomaw (@JacobTomaw) August 26, 2015

I found instructions to use Snap CI to do deployments. I also discovered that Travis CI can do the same thing.

I now have a very simple pipeline in Snap CI that will deploy to S3. The important bit is the command that the stage will run. 

aws s3 sync . s3://tomawtech.com --exclude .gitignore --exclude README.md --acl public-read --region us-east-1 --delete
Snap CI put in some good defaults, but it was still useful to check the AWS S3 sync documentation. The pipeline is setup to run with every commit and I will be notified if something bad ever happens. I can also see the status on the site's repository home page.

In other news I added bootstrap to the site with united theme from Bootswatch. I still have a lot to learn about Bootstrap.

Welcome to my new site!

I am trying to set up a new blog site from scratch to help learn more about the AWS service offerings and to share what I know and how I can help IT organizations. I want to document the process as much as possible. There is going to be a lot of churn at first.

To start with I have followed the Setting Up a Static Website Using a Custom Domain walk-through at AWS. I registered tomawtech.com with Namecheap. I have also setup dandelion to synchronize the Github repository that holds the source for this site.

Here is the example dandelion.yml file. I do not keep the actual dandelion file in Github because it has my AWS Secret Key in it. I am investigating how to keep this stored encrypted in plain sight.

I also had to move my nameserver over to Route 53 by creating a new Hosted Zone. This allowed me to learn a lot about name records than I knew before.
